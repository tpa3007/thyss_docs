## Обзор

Интеграция THYSS × Tilda позволяет принимать USDT-платежи прямо через формы и корзину Tilda. Покупатель оформляет заказ привычным способом, THYSS автоматически создаёт страницу оплаты и конвертирует сумму из рублей в USDT.

```
┌─────────────────────────────────────────────────────────────┐
│                        ПОКУПАТЕЛЬ                           │
└──────────────────────────┬──────────────────────────────────┘
                           │ нажимает «Оплатить»
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                          TILDA                              │
│       Собирает данные формы → POST → api.thyss.xyz          │
└──────────────────────────┬──────────────────────────────────┘
                           │ данные заказа + подпись
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                        THYSS API                            │
│    Проверяет подпись → создаёт платёжную сессию             │
│    Конвертирует RUB → USDT по текущему курсу (+1% буфер)    │
└──────────────────────────┬──────────────────────────────────┘
                           │ redirect → pay.thyss.xyz
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                   СТРАНИЦА ОПЛАТЫ (HPP)                     │
│          QR-код · TON-адрес · таймер 30 минут               │
└──────────────────────────┬──────────────────────────────────┘
                           │ покупатель платит через Tonkeeper
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                    АВТОПОДТВЕРЖДЕНИЕ                         │
│    THYSS обнаруживает платёж (1–2 мин) → статус: Success    │
│    Webhook на ваш сервер (если настроен)                     │
└─────────────────────────────────────────────────────────────┘
```

## Требования

| # | Требование | Где проверить |
|---|---|---|
| 1 | Аккаунт THYSS зарегистрирован | dashboard.thyss.xyz |
| 2 | TON-кошелёк подключён | THYSS → Настройки → Кошелёк |
| 3 | Сайт Tilda с корзиной или формой оплаты | Редактор Tilda |
| 4 | Public Key получен | THYSS → Настройки → Интеграция → Tilda |
| 5 | Secret Key получен | THYSS → Настройки → Разработчикам |

---

## Шаг 1. Получите ключи в THYSS

### Public Key и Endpoint URL

Откройте: **THYSS Дашборд → Настройки → Интеграция → вкладка «Tilda»**

Вы увидите готовый Endpoint URL:
```
https://api.thyss.xyz/integrations/tilda?pk=pk_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

Нажмите «Копировать» — он понадобится на шаге 3.

<Note>
  Public Key (часть после `?pk=`) — публичный идентификатор. Его безопасно вставлять в URL.
</Note>

### Secret Key

Откройте: **THYSS Дашборд → Настройки → Разработчикам**

Найдите поле **Secret Key**:
```
sk_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

<Warning>
  Secret Key — приватный. Никому не передавайте его и не публикуйте. При подозрении на утечку — ротируйте через кнопку «Сгенерировать новый».
</Warning>

---

## Шаг 2. Создайте платёжную систему в Tilda

1. Войдите в Tilda и откройте нужный проект
2. Перейдите в раздел **«Оплата»**
3. Нажмите **«Добавить платёжную систему»**
4. Выберите **«Другие»** → **«Универсальная платёжная система»**
5. Выберите **«Новая платёжная система (для разработчиков)»**

Откроется страница конфигурации — заполняйте поля строго по инструкции ниже.

---

## Шаг 3. Основные настройки

### Логин
Вставьте ваш **Public Key**:
```
pk_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### Секрет для подписи заказа
Вставьте ваш **Secret Key**:
```
sk_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### URL для уведомлений
Оставьте **пустым** — THYSS самостоятельно отслеживает платежи.

### Валюта

| Магазин в... | Укажите |
|---|---|
| Рублях | `RUB` |
| Долларах | `USD` |
| USDT напрямую | `USDT` |

### Страна и Язык виджета
Не трогайте — THYSS не использует эти поля.

---

## Шаг 4. API URL

Раскройте **«Расширенные настройки»** → подраздел **«URL»**.

В поле **API URL** вставьте Endpoint URL из шага 1:
```
https://api.thyss.xyz/integrations/tilda?pk=pk_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

<Warning>
  Убедитесь, что в URL подставлен **ваш** `pk_live_...`. URL без параметра `?pk=` работать не будет.
</Warning>

Раздел **«Поля»** → **«Список соответствия полей»**.

В каждой строке есть колонка **«Поле»** — вписывайте значения из таблицы точно:

| Строка в Tilda | Вписать в «Поле» | Обязательно |
|---|---|---|
| Логин | `login` | ✅ |
| Валюта | `currency` | ✅ |
| Номер заказа | `orderid` | ✅ |
| Сумма заказа | `payment` | ✅ |
| Подпись | `secret` | ✅ |
| Email покупателя | `Email` | Рекомендуется |
| Телефон покупателя | `Phone` | Рекомендуется |
| Имя покупателя | `Name` | Рекомендуется |
| Товары | `products` | При наличии корзины |
| Язык | *(пусто)* | — |
| Страна | *(пусто)* | — |
| URL для уведомлений | *(пусто)* | — |
| Описание заказа | *(пусто)* | — |
| Время заказа | *(пусто)* | — |

<Note>
  Email, Phone, Name не влияют на обработку платежа, но заполняют карточку клиента в THYSS → Клиенты.
</Note>

## Шаг 7. Настройка подписи заказа

<Warning>
  Это самый критичный раздел. Ошибки здесь — главная причина ошибки «Invalid signature». Заполняйте строго по таблице.
</Warning>

| Поле | Значение |
|---|---|
| Правила подписи | `Все поля` |
| Исключить поля | `secret` |
| Порядок сортировки полей | `В алфавитном порядке, с начала` |
| Разделитель | *(пусто)* |
| **Добавлять секрет в подпись** | **`Последним элементом`** |
| **Алгоритм шифрования подписи** | **`SHA-256`** ← не MD5! |
| Использовать секрет как ключ HMAC | ❌ Выключено |
| Преобразовать к верхнему регистру | ❌ Выключено |
| Преобразовать подпись в base64 | ❌ Выключено |

<Warning>
  По умолчанию в Tilda стоит **MD5** — его **обязательно** нужно изменить на **SHA-256**. Это самая распространённая ошибка при настройке.
</Warning>

## Шаг 10. Разрешённые домены в THYSS

Вернитесь в **THYSS → Настройки → Интеграция → Tilda**.

В поле **«Разрешённые домены»** укажите домен вашего сайта:

```
myshop.ru
```

Несколько сайтов — каждый с новой строки:
```
myshop.ru
shop.myshop.ru
```

<Warning>
  Указывайте только домен, без `https://` и слеша. Поле обязательно — без домена все запросы от Tilda будут заблокированы.
</Warning>

## Шаг 11. Публикация и тест

<Steps>
  <Step title="Опубликуйте сайт в Tilda">
    Изменения в настройках платёжной системы применяются только после публикации. Нажмите **«Опубликовать»** в редакторе.
  </Step>
  <Step title="Сделайте тестовый заказ">
    Откройте опубликованный сайт, добавьте товар, заполните форму, нажмите «Оплатить». Браузер должен перенаправить вас на `pay.thyss.xyz` со страницей оплаты.
  </Step>
  <Step title="Проверьте в THYSS">
    Перейдите в **THYSS → Платежи**. Должна появиться транзакция со статусом `Pending` и источником `tilda`.
  </Step>
</Steps>

<Note>
  Статус заказа в Tilda-кабинете останется «Ожидает оплаты» даже после успешного платежа — это ожидаемое поведение. Tilda не получает callback от THYSS. Актуальный статус всегда в **THYSS → Платежи**.
</Note>

## Часто задаваемые вопросы

<AccordionGroup>
  <Accordion title="Ошибка «Invalid signature» при оформлении заказа">
    Проверьте по порядку:
    1. Алгоритм шифрования — должен быть `SHA-256`, не MD5
    2. «Добавлять секрет в подпись» — должно быть `Последним элементом`, не `Нет`
    3. Чекбокс HMAC — должен быть **выключен**
    4. Поле «Исключить поля» — должно содержать `secret`
  </Accordion>

  <Accordion title="Браузер показывает JSON вместо страницы оплаты">
    Используйте стандартный блок корзины Tilda (ST100). THYSS автоматически определяет браузерные запросы и возвращает redirect на pay.thyss.xyz.
  </Accordion>

  <Accordion title="Ошибка «Invalid origin» / заблокированный домен">
    Ваш домен не добавлен в поле «Разрешённые домены» в THYSS, или указан с `https://`. Формат: `myshop.ru` (без протокола).
  </Accordion>

  <Accordion title="Покупатель заплатил, но в Tilda статус «Ожидает»">
    Это ожидаемое поведение — Tilda не получает callback от THYSS. Реальный статус платежа всегда в **THYSS → Платежи**.
  </Accordion>

  <Accordion title="В какой валюте принимается оплата?">
    Всегда в **USDT (TON-сеть)**. Если заказ в рублях — THYSS конвертирует по актуальному курсу с буфером +1%. Курс фиксируется на 30 минут.
  </Accordion>

  <Accordion title="Можно подключить несколько Tilda-сайтов?">
    Да. Один Endpoint URL работает для всех сайтов. Добавьте все домены в поле «Разрешённые домены» (по одному на строку). Платежи поступают на один кошелёк.
  </Accordion>

  <Accordion title="Что происходит, если покупатель не заплатил за 30 минут?">
    Сессия истекает. Для повторной оплаты нужно оформить заказ заново.
  </Accordion>
</AccordionGroup>

## Техническая справка

<Note>
  Этот раздел для разработчиков. При стандартной настройке через Tilda он не нужен.
</Note>

### Алгоритм проверки подписи

```
1. Взять все поля POST-запроса, кроме поля "secret"
2. Отсортировать по имени — лексикографически (ASCII):
   заглавные буквы A–Z идут ПЕРЕД строчными a–z
   Пример: Email, Name, Phone, Receipt, currency, login, orderid, payment
3. Взять только значения → склеить без разделителя
4. Добавить Secret Key в конец строки
5. Вычислить SHA-256 → результат в lowercase hex
6. Сравнить с полем "secret" из запроса
```

<Warning>
  THYSS использует **лексикографическую** сортировку (ASCII), а не `localeCompare`. Заглавные поля (`Email`, `Name`, `Phone`) идут **перед** строчными (`currency`, `login`, `orderid`). Это критично для правильного вычисления подписи.
</Warning>

### Структура POST-запроса от Tilda

**Метод:** `POST`
**Content-Type:** `application/x-www-form-urlencoded`
**URL:** `https://api.thyss.xyz/integrations/tilda?pk={ваш_public_key}`

| Поле | Тип | Описание |
|---|---|---|
| `orderid` | string | Номер заказа в Tilda |
| `payment` | string | Сумма заказа |
| `currency` | string | Валюта (`RUB`, `USD`, `USDT`) |
| `secret` | string | SHA-256 подпись |
| `login` | string | Public Key мерчанта |
| `test` | string | `"1"` в тестовом режиме |
| `Name` | string | Имя покупателя |
| `Email` | string | Email покупателя |
| `Phone` | string | Телефон покупателя |
| `products` | JSON string | Массив товаров корзины |

### Ответ THYSS

**Браузерный запрос (redirect):**
```http
HTTP/1.1 302 Found
Location: https://pay.thyss.xyz?cs=cs_live_...
```

**API-запрос (JSON):**
```json
{
  "status": "ok",
  "payment": {
    "orderid": "23969993:1234567890",
    "url": "https://pay.thyss.xyz?cs=cs_live_..."
  }
}
```

**Ошибка:**
```json
{
  "status": "error",
  "message": "Invalid signature"
}
```
