## Ошибка «Invalid signature»

Самая частая проблема при интеграции с Tilda. Проверяйте по порядку:

<Steps>
  <Step title="Алгоритм подписи">
    В Tilda → раздел «Подпись заказа» → поле «Алгоритм шифрования» должно быть **SHA-256**, не MD5.

    MD5 стоит по умолчанию — это самая распространённая ошибка.
  </Step>
  <Step title="Добавление секрета">
    Поле «Добавлять секрет в подпись» должно быть **«Последним элементом»**, не «Нет».
  </Step>
  <Step title="HMAC выключен">
    Чекбокс «Использовать секрет в качестве ключа алгоритма (HMAC)» должен быть **снят**.
  </Step>
  <Step title="Исключение поля secret">
    В поле «Исключить поля» должно быть `secret`. Не `signature`, не пустое значение — именно `secret`.
  </Step>
  <Step title="Правильный Secret Key">
    Убедитесь, что в поле «Секрет для подписи заказа» вставлен **Secret Key** (`sk_live_...`), а не Public Key.
  </Step>
</Steps>

## Браузер показывает JSON вместо страницы оплаты

**Симптом:** После нажатия «Оплатить» браузер переходит на `api.thyss.xyz/...` и отображает текст вида `{"status":"ok","payment":{...}}`.

**Причина:** Tilda использует нестандартный тип блока оплаты, который не отправляет `Accept: text/html` заголовок.

**Решение:** Переключитесь на стандартный блок корзины Tilda (ST100). THYSS автоматически определяет браузерные запросы по заголовку `Accept` и возвращает redirect на pay.thyss.xyz.

## Ошибка «Invalid origin» / домен заблокирован

**Симптом:** Платёж не создаётся, ошибка связана с доменом.

**Проверьте:**

1. Открыт **THYSS → Настройки → Интеграция → Tilda**
2. В поле «Разрешённые домены» есть ваш домен
3. Домен указан **без** `https://` и слеша

```
✅ Правильно:   myshop.ru
❌ Неправильно: https://myshop.ru
❌ Неправильно: myshop.ru/
```

## Покупатель заплатил, но статус в Tilda «Ожидает»

**Это ожидаемое поведение.** Tilda не получает обратный callback от THYSS — статус в её кабинете не обновляется автоматически.

Реальный статус платежа всегда смотрите в **THYSS → Платежи**.

---

## Платёж завис в статусе «Pending»

Если транзакция висит в `Pending` дольше 5 минут:

1. Убедитесь, что покупатель отправил точную сумму USDT (включая буфер +1%) на правильный адрес
2. Покупатель должен указать **Memo** (комментарий) к переводу — без него платёж не будет сопоставлен
3. Если покупатель забыл указать memo — используйте **THYSS → Платежи → Разрешить вручную**

## Ошибка «Monthly transaction limit reached»

Превышен лимит тарифа START (10 транзакций в месяц).

Для снятия ограничения перейдите на тариф **BUSINESS**: THYSS → Настройки → Тариф.

---

## Ошибка «Merchant wallet not configured»

В аккаунте THYSS не указан TON-кошелёк. Откройте **Настройки → Кошелёк** и добавьте адрес.
